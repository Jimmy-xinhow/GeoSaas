# ADR-004：平台文章與官網第一方內容分離

- 狀態：Accepted
- 日期：2026-07-17

## 背景

Geovault 平台文章與客戶官方網站文章的發布目的、canonical、權限與內容來源不同。直接把平台文章全文匯出給客戶會造成跨網域重複內容，也可能使平台與客戶官網互相競爭索引。

## 決策

新增獨立的 `OfficialSiteArticle` 模型與 `/sites/:siteId/official-articles` API。生成服務只使用平台文章 metadata 加上客戶第一方資料，不傳入平台正文；所有候選文章必須通過相似度與品質閘門，並經客戶核准後才可取得官網內容包。

## 取捨

- 優點：資料層隔離降低誤發布風險；每週平台生成保持原流程；可追蹤官網文章版本與上線驗證。
- 成本：需要新增資料表、API、前端工作區與審核步驟；客戶仍需在自己的 CMS 完成上架。
- 未來：可在客戶明確授權後加入 WordPress/Webflow connector，但不得繞過現有品質與相似度閘門。

## 失敗處理

第一方資料不足、AI 格式錯誤、相似度過高或正式網址驗證失敗時，文章保持不可匯出的狀態，不影響 `client_daily` 平台文章排程。
